砲彈射擊

  • 製作「射擊」遊戲，在學習Scratch的過程中是一定會遇到的。
  • 「射擊」動作要完成，除了要有觸發事件外，還得有二個物件，其一是如槍身、砲管...等，用來進行發射的主體，其二就是子彈、砲彈...等。
  • 通常以「xxx是否被按下？」、「xxx是否被點擊？」的〖布林積木〗來作為觸發條件。
  • 射擊可以是單發射擊，也可以是連續射擊。當是連續射擊時，該角色將會建立分身。
  • 我們以<(空白 v)鍵被按下？>指定空白鍵或< 滑鼠鍵被按下？>來做為觸發事件的條件判斷。
  • 將「射擊」區分為每次發射都要啟動一次觸發器的半自動射擊（一次只能發射一顆），以及按住觸發器不放開，就可以持續發射的自動射擊。
  1. 二角色，皆不用分身
    1. 半自動射擊
    2. 自動射擊
  2. 二角色，其中一個建分身
    1. 半自動射擊
    2. 自動射擊
  3. 一角色，用本尊及建自己分身
    1. 半自動射擊
    2. 自動射擊
二角色，皆不用分身

這種方式需要二個角色，一個是發射的主體，另一個是飛射出的子彈或砲彈，都不使用分身。

📍半自動射擊

此腳本要求玩家要再次射擊時，得放開按建，再次點擊才能生效。

											當 @greenflag 被點擊 // 槍管角色
											重複無限次
											等待直到 <<(空白 v)鍵被按下？> 或 <滑鼠鍵被按下？>>
											廣播訊息 (shoot v) 並等待
											等待直到 <not <<(空白 v)鍵被按下？> 或 <滑鼠鍵被按下？>>> // 一次只能發射一顆
											end
											
											當收到訊息 [shoot v] //子彈角色
											定位到 (槍管 v)位置 //從槍管位置發射
											顯示
											重複直到 <<碰到 (目標物 v)？> 或 <碰到 (邊緣 v)？>> //移動到會令子彈消失處為止
											移動 (10) 點
											end
											隱藏

💥一次僅能發射一顆，無法快速射擊。

📍自動射擊

此腳本可以使玩家只要按住按鍵就能一直發射。

											當 @greenflag 被點擊 //槍管角色
											重複無限次
											等待直到 <<(空白 v)鍵被按下？> 或 <滑鼠鍵被按下？>>
											廣播訊息 (shoot v) 並等待
											等待 (0.1) 秒 // 不用放開按鍵，每0.1秒就廣播一次shoot訊息
											end
											
											當收到訊息 [shoot v] //子彈角色
											定位到 (槍管 v)位置
											顯示
											重複直到 <<碰到 (目標物 v)？> 或 <碰到 (邊緣 v)？>>
											移動 (10) 點
											end
											隱藏
二角色，其中一個建分身

這種方式其中一個角色（子彈）是以分身來顯示，因此，在舞臺上是允許多顆子彈同時存在。

📍半自動射擊

此腳本要求玩家要再次射擊時，得放開按建，再次點擊才能生效。

											當 @greenflag 被點擊 // 槍管角色
											重複無限次
											等待直到 <<(空白 v)鍵被按下？> 或 <滑鼠鍵被按下？>>
											建立 (子彈 v)的分身 // 建立子彈角色的分身
											等待直到 <<<(空白 v)鍵被按下？> 或 <滑鼠鍵被按下？>>不成立>
											end
											
											當分身產生 // 子彈角色
											重複直到 <<碰到 (目標物 v)？> 或 <碰到 (邊緣 v)？>>
											移動 (10) 點
											end
											分身刪除

📍自動射擊

此腳本可以連續發射子彈。

											當 @greenflag 被點擊 // 槍管角色
											重複無限次
											等待直到 <<(空白 v)鍵被按下？> 或 <滑鼠鍵被按下？>>
											建立 (子彈 v)的分身
											等待 (0.1) 秒 // 每次子彈發射的間隔秒數
											end
											
											當分身產生 // 子彈
											重複直到 <<碰到 (目標物 v)？> 或 <碰到 (邊緣 v)？>>
											移動 (10) 點
											end
											分身刪除
一角色，用本尊及建自己分身

這種方式更有效率，只要一個具有二種造型的角色。
